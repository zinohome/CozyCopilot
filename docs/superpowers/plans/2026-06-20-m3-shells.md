## 0. Goal

M2 closed the BFF coverage gap. The web bundle is now feature-complete from the CozyEngineV2 spec's standpoint, but it can only run in a browser. M3 wraps that bundle in:

- **Tauri 2.x** desktop shell (macOS + Windows)
- **Capacitor 7.x** mobile shell (iOS + Android)

…with a **capability abstraction** (`lib/capabilities`) so business code stays platform-agnostic. M3 also adds:
- **OS notification** abstraction (`lib/notifications`) — Tauri uses native notifications, Capacitor uses Capacitor LocalNotifications, web uses Notification API
- **Unified storage** (`lib/storage`) — web=localStorage, Tauri=tauri-plugin-store, Capacitor=@capacitor/preferences
- **Microphone permission UX** — capability detection + permission request flow

After M3, the same Next.js bundle runs on **4 surfaces** (own-hosted Web, embed widget, Tauri desktop, Capacitor mobile) with **zero platform-specific code outside `lib/capabilities`** and `lib/notifications`.

## 2. Architecture

### 2.1 Capability abstraction

The capability layer is a **runtime polymorphism** pattern: a single `index.ts` exports a unified API; the implementation is chosen at import time based on `typeof window !== "undefined"` and platform globals.

```typescript
// src/lib/capabilities/index.ts
import * as web from "./web";
import * as tauri from "./tauri";
import * as capacitor from "./capacitor";

const impl = (() => {
  if (typeof window === "undefined") return web;          // SSR → web fallback
  // @ts-expect-error — tauri injects __TAURI_INTERNALS__ at runtime
  if (window.__TAURI_INTERNALS__) return tauri;
  // @ts-expect-error — capacitor injects Capacitor at runtime
  if (window.Capacitor) return capacitor;
  return web;
})();

export const checkMicrophonePermission = impl.checkMicrophonePermission;
export const requestMicrophonePermission = impl.requestMicrophonePermission;
export const isNativeApp = impl.isNativeApp;
export const getPlatform = impl.getPlatform;
```

```typescript
// src/lib/capabilities/web.ts
export async function checkMicrophonePermission(): Promise<PermissionState> {
  if (!navigator.permissions) return "prompt";
  const result = await navigator.permissions.query({ name: "microphone" as PermissionName });
  return result.state;
}

export async function requestMicrophonePermission(): Promise<boolean> {
  try {
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    stream.getTracks().forEach(t => t.stop());
    return true;
  } catch {
    return false;
  }
}

export const isNativeApp = false;
export const getPlatform = (): "web" | "embed" => {
  if (typeof window !== "undefined" && window.self !== window.top) return "embed";
  return "web";
};
```

```typescript
// src/lib/capabilities/tauri.ts
import { invoke } from "@tauri-apps/api/core";

export async function checkMicrophonePermission(): Promise<PermissionState> {
  // Tauri 2.x uses plugins; v1.0 we trust OS-level prompt
  return "prompt";
}

export async function requestMicrophonePermission(): Promise<boolean> {
  // Tauri's webview delegates to the web `navigator.mediaDevices.getUserMedia`
  return web.requestMicrophonePermission();
}

export const isNativeApp = true;
export const getPlatform = (): "tauri-mac" | "tauri-win" | "tauri-linux" => {
  // Tauri exposes `window.__TAURI_INTERNALS__.platform` or similar
  // For v1.0, just default to current OS via navigator
  const ua = typeof navigator !== "undefined" ? navigator.platform : "";
  if (ua.startsWith("Mac")) return "tauri-mac";
  if (ua.startsWith("Win")) return "tauri-win";
  return "tauri-linux";
};
```

```typescript
// src/lib/capabilities/capacitor.ts
import { Capacitor } from "@capacitor/core";
import { Permissions } from "@capacitor/permissions";  // M3 plugin

export async function checkMicrophonePermission(): Promise<PermissionState> {
  const result = await Permissions.checkPermissions();
  // @ts-expect-error — Capacitor permission type union is large
  return result.microphone ?? "prompt";
}

export async function requestMicrophonePermission(): Promise<boolean> {
  const result = await Permissions.requestPermissions({ permissions: ["microphone"] });
  // @ts-expect-error — same
  return result.microphone === "granted";
}

export const isNativeApp = true;
export const getPlatform = (): "ios" | "android" => {
  return Capacitor.getPlatform() as "ios" | "android";
};
```

### 2.2 Notification abstraction

```typescript
// src/lib/notifications/index.ts
import * as web from "./web";
import * as tauri from "./tauri";
import * as capacitor from "./capacitor";

const impl = /* same runtime dispatch as capabilities */;
export const notify = impl.notify;
export const requestPermission = impl.requestPermission;
```

```typescript
// src/lib/notifications/web.ts
export async function requestPermission(): Promise<boolean> {
  if (!("Notification" in window)) return false;
  if (Notification.permission === "granted") return true;
  if (Notification.permission === "denied") return false;
  const result = await Notification.requestPermission();
  return result === "granted";
}

export async function notify(opts: { title: string; body?: string; icon?: string }): Promise<void> {
  if (Notification.permission !== "granted") return;
  new Notification(opts.title, { body: opts.body, icon: opts.icon });
}
```

```typescript
// src/lib/notifications/tauri.ts
import { sendNotification, isPermissionGranted, requestPermission } from "@tauri-apps/plugin-notification";

export async function requestPermission(): Promise<boolean> {
  if (await isPermissionGranted()) return true;
  const result = await requestPermission();
  return result === "granted";
}

export async function notify(opts: { title: string; body?: string }): Promise<void> {
  if (!(await isPermissionGranted())) return;
  sendNotification({ title: opts.title, body: opts.body });
}
```

```typescript
// src/lib/notifications/capacitor.ts
import { LocalNotifications } from "@capacitor/local-notifications";

export async function requestPermission(): Promise<boolean> {
  const result = await LocalNotifications.requestPermissions();
  return result.display === "granted";
}

export async function notify(opts: { title: string; body?: string; id?: number }): Promise<void> {
  await LocalNotifications.schedule({
    notifications: [{
      title: opts.title,
      body: opts.body,
      id: opts.id ?? Math.floor(Math.random() * 100000),
    }],
  });
}
```

### 2.7 React hooks

```typescript
// src/hooks/useCapability.ts
import { useEffect, useState } from "react";
import * as cap from "@/lib/capabilities";

export function useCapability(name: "mic"): { state: PermissionState; request: () => Promise<boolean> } {
  const [state, setState] = useState<PermissionState>("prompt");
  useEffect(() => {
    cap.checkMicrophonePermission().then(setState);
  }, []);
  return {
    state,
    request: async () => {
      const ok = await cap.requestMicrophonePermission();
      setState(ok ? "granted" : "denied");
      return ok;
    },
  };
}

export function usePlatform(): { isNativeApp: boolean; platform: string } {
  return { isNativeApp: cap.isNativeApp, platform: cap.getPlatform() };
}
```

```typescript
// src/hooks/useNotify.ts
import { useEffect, useState } from "react";
import * as notif from "@/lib/notifications";

export function useNotify() {
  const [permission, setPermission] = useState<NotificationPermission | "unsupported">("default");
  useEffect(() => {
    notif.requestPermission().then(ok => setPermission(ok ? "granted" : "denied"));
  }, []);
  return {
    permission,
    notify: notif.notify,
  };
}
```

### 2.8 Mic permission UX

```typescript
// src/components/mic/MicPermissionPrompt.tsx
"use client";
import { useState } from "react";
import { useCapability } from "@/hooks/useCapability";
import { Button } from "@/components/ui/button";

export function MicPermissionPrompt({ onGranted }: { onGranted: () => void }) {
  const { state, request } = useCapability("mic");
  const [requesting, setRequesting] = useState(false);

  if (state === "granted") {
    onGranted();
    return null;
  }

  return (
    <div className="rounded-md border border-[var(--border)] bg-[var(--surface)] p-4">
      <h3 className="text-sm font-medium">麦克风权限</h3>
      <p className="mt-1 text-xs text-[var(--text-muted)]">
        {state === "denied"
          ? "麦克风权限被拒绝。请在浏览器/系统设置中允许后重试。"
          : "需要麦克风权限才能使用语音输入。"}
      </p>
      <Button
        onClick={async () => {
          setRequesting(true);
          await request();
          setRequesting(false);
        }}
        disabled={requesting || state === "denied"}
        className="mt-3"
      >
        {state === "denied" ? "已拒绝" : "允许麦克风"}
      </Button>
    </div>
  );
}
```

### M3.6 — Update auth store to use `lib/storage`
**Files:** `src/stores/auth.ts` (modify)

Replace `createJSONStorage(() => localStorage)` with `createJSONStorage(() => makeStorage())` where `makeStorage()` is a small adapter that returns a Storage-like object backed by `lib/storage.getItem/setItem`.

**Risk:** Zustand's `createJSONStorage` expects a synchronous `Storage` (with `getItem/setItem/removeItem` returning strings, not Promises). `lib/storage` is async. Need a small sync shim for auth (since we only need localStorage-like behavior on web; on Tauri/Capacitor, tauri-plugin-store and Capacitor preferences are also async but Zustand hydration is best-effort).

**Two options:**
- (a) Use `lib/storage` async, and Zustand's `skipHydration: true` + manual rehydrate after the first async read
- (b) Keep direct `localStorage` for auth and add `lib/storage` for everything else (M4+)

**Recommendation:** (a) — it's cleaner and M3 is the right time to do it.

## 5. Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Tauri 2.x permissions API is recent (still maturing) | Med | Med | Use the latest stable tauri 2.x; pin exact version |
| `tauri-plugin-store` async API doesn't fit Zustand's sync `Storage` interface | High | Med | Use `skipHydration: true` + manual rehydrate pattern (M3.6 option a) |
| Capacitor `Info.plist` / `AndroidManifest.xml` permission strings must be set BEFORE first build | Med | High | Add to `capacitor.config.ts` and verify with `npx cap sync` |
| `pnpm build:web:static` may not produce output that Tauri/Capacitor can load (e.g., BFF routes need runtime) | Med | High | Tauri/Capacitor load via `tauri://`/`capacitor://` scheme; the BFF routes still work as dynamic within the bundle (per M1.4 verification) |
| `next.config.ts` `output: "export"` conflict with SSR build | Low | Med | Use env var to switch modes, not direct config change |
| Cargo / Rust toolchain not installed in dev environment | High | Low | Document in README; gate `pnpm build:desktop` behind `which cargo` check |
| Xcode / Android Studio not installed in dev environment | High | Low | Document; gate mobile build behind platform checks |
