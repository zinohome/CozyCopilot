# Android Shell (Capacitor 7.x)

Hand-scaffolded by M3.9 — the real Gradle / Android Studio project is
auto-generated by `npx cap add android`. This directory exists so:

1. **`AndroidManifest.xml` carries the Android runtime permissions
   CozyCopilot needs.** `RECORD_AUDIO` is required for voice chat
   (`src/lib/capabilities/`), `POST_NOTIFICATIONS` is required for
   `LocalNotifications.schedule(...)` (see M3.3 +
   `src/lib/notifications/capacitor.ts`). Without these, Android
   silently denies the corresponding API call.
2. **The package id matches `capacitor.config.ts`**
   (`com.zinohome.cozycopilot`).
3. **The minimum expected file layout is present** so the next agent
   that runs `npx cap add android` finds an empty-but-correct tree
   and only needs to fill in the auto-generated parts.

## Re-generating the Gradle project

```bash
# Requires Android Studio + Android SDK (compileSdk 35) + JDK 17
cd /path/to/CozyCopilot
npx cap add android        # creates android/app/build.gradle, settings.gradle, etc.
npx cap sync android       # copies out/ → android/app/src/main/assets/public
npx cap open android       # opens the project in Android Studio
```

Then in Android Studio: pick a device (Pixel emulator or physical
device), and `Run`. The Gradle wrapper, `gradle.properties`, and
`local.properties` are all created by the first `npx cap add android`
run.

## Permissions currently declared

- `INTERNET` — basic network for the Next.js BFF (`fetch` calls).
- `RECORD_AUDIO` — voice chat in
  `src/features/voice/MicPermissionPrompt.tsx`. Runtime permission
  (API 23+); prompt the user via the abstraction in
  `src/lib/capabilities/`.
- `POST_NOTIFICATIONS` — required for `LocalNotifications.schedule`
  to display anything on Android 13+ (API 33+).
- `SCHEDULE_EXACT_ALARM` / `USE_EXACT_ALARM` — for time-precise
  scheduled notifications.
- `RECEIVE_BOOT_COMPLETED` / `VIBRATE` / `WRITE_SETTINGS` — required
  by `@capacitor/local-notifications` on Android.

## What was NOT verified locally

- `gradlew assembleDebug` not run (Android SDK not installed in this env).
- No device build / no emulator screenshot.

These checks happen on a real Android dev machine during the M3
wrap-up (M3.11) or in CI.
