# iOS Shell (Capacitor 7.x)

Hand-scaffolded by M3.9 — the real `.xcodeproj` is auto-generated by
`npx cap add ios`. This directory exists so:

1. **`Info.plist` carries the iOS privacy keys CozyCopilot needs.**
   `NSMicrophoneUsageDescription` is required because the app's voice
   chat calls `getUserMedia({ audio: true })` through the
   `src/lib/capabilities/` abstraction (see M3.1 / M3.2). Without
   this key, iOS silently denies the prompt and the user has no idea
   why the app cannot hear them.
2. **`capacitor.config.ts` at the repo root points to `out/` (the
   `next build` static export) as `webDir`.** Capacitor's
   `npx cap sync` copies that directory into `ios/App/App/public/`
   on the next run.
3. **The minimum expected file layout is present**, so the next agent
   that runs `npx cap add ios` finds an empty-but-correct tree and
   only needs to fill in the auto-generated parts.

## Re-generating the Xcode project

```bash
# Requires Xcode 15+ and CocoaPods 1.13+
cd /path/to/CozyCopilot
npx cap add ios            # creates ios/App.xcodeproj + App target
npx cap sync ios           # copies out/ → ios/App/App/public, refreshes Pods
npx cap open ios           # opens the project in Xcode
```

Then in Xcode: select a Team under Signing & Capabilities, choose a
real device or generic iOS device, and `Cmd-R` to build. The
`LaunchScreen.storyboard` and `Assets.xcassets/` placeholders come
from the auto-generated tree; ship real app icons in
`ios/App/App/Assets.xcassets/AppIcon.appiconset/` before submitting to
the App Store.

## Permissions currently declared

- `NSMicrophoneUsageDescription` — voice chat in
  `src/features/voice/MicPermissionPrompt.tsx`. Add more keys here
  when new native capabilities are introduced (camera, photo library,
  push notifications).
